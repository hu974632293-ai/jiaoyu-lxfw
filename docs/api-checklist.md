## 3. 生产级 API 验收要求

1. 所有 API 统一返回 `{ code, msg, data }`，错误也使用同一结构。
2. 写操作必须经过 service 层规则校验，不在路由中堆业务逻辑。
3. 有状态流转的对象必须记录处理人、处理时间、处理结果和历史记录。
4. 多角色对象必须同时验证发起方和处理方视角。
5. 列表接口应支持必要筛选、空状态和分页或数量控制。
6. 删除优先采用归档或软删除；如确需硬删除，必须明确数据影响范围。
7. 公开官网接口不得返回内部 CRM、审计、权限、seed、fallback 或调试信息。
8. 普通业务页面使用的接口不得要求前端拼接复杂业务规则。
