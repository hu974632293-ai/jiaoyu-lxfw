# 客户增长与客户 360 前端信息架构设计

## 1. 定位

客户增长不再代表整个后台，也不再和“二期助手”并列组织全系统能力。它是顾问和管理者使用的客户业务域，核心目标是把客户需求表中的“客户研判”和客服 Agent 沉淀的线索转化为可跟进、可推荐、可报名、可报告的客户闭环。

## 2. 使用角色

| 角色 | 权限范围 |
| --- | --- |
| 顾问 | 新建线索、上传资料、触发研判、跟进、任务、阶段流转、客户 360 |
| 管理者 | 查看客户经营总览、重点客户、漏斗、报告和风险 |
| 管理员 | 系统治理需要时可查看数据和配置 |
| 员工 | 仅在员工工作台中做客户快捷录入/查询/状态更新，不默认进入完整客户增长 |

## 3. 顾问客户增长页

页面目标：顾问进入后可以立即处理客户，而不是只看列表。

必备区域：

| 区域 | 功能 |
| --- | --- |
| 主操作区 | 新建线索、上传资料、粘贴资料、触发研判 |
| 漏斗 | 新线索、已研判、咨询中、活动邀约、成交、流失 |
| 筛选 | 关键词、阶段、负责人、推荐项目、创建时间 |
| 客户列表 | 客户、阶段、推荐项目、负责人、最近跟进、下一步 |
| 快速操作 | 新增跟进、创建任务、更新阶段、进入客户 360 |

强制规则：

1. `新建线索` 必须是显著主按钮。
2. 不允许只依赖官网咨询或 seed 数据产生客户。
3. 顾问无权限的系统治理、员工日报、学生心理明细不展示。
4. 页面不写长解释文字。

## 4. 客户 360

客户 360 是单个客户的详情工作区，不作为普通一级导航。

Tabs：

| Tab | 内容 |
| --- | --- |
| 概览 | 基本信息、阶段、负责人、下一步动作 |
| 画像研判 | 规则命中、匹配度、缺失字段、风险提示 |
| 推荐项目 | 推荐项目、推荐理由、适合人群、项目详情 |
| 咨询记录 | 官网客服 Agent、Dify 问答、来源引用 |
| 跟进任务 | 跟进记录、待办任务、阶段历史、成交/流失 |
| 活动报名 | 推荐活动、报名记录、签到结果 |
| 报告快照 | 客户相关经营建议和历史报告 |

## 5. 管理者客户经营视图

管理者不需要顾问式操作密度，应优先看指标和风险。

区域：

- 新增线索趋势。
- 高潜客户。
- 成交/流失。
- 超时未跟进。
- 活动转化。
- 客户经营报告入口。

## 6. 数据和 API

| 功能 | API | 数据表 |
| --- | --- | --- |
| 新建线索 | `POST /api/leads` | `lead`, `customer` |
| 上传/粘贴资料 | `POST /api/leads/{id}/source` | `lead_source_file` |
| 画像研判 | `POST /api/profile/assess` | `lead_profile_assessment`, `profile_rule_hit` |
| 推荐项目 | `GET /api/leads/{id}/recommendations` | `lead_recommendation`, `course_project` |
| 跟进记录 | `POST /api/leads/{id}/follow-ups` | `lead_follow_up` |
| 任务 | `POST /api/leads/{id}/tasks` | `lead_task` |
| 阶段流转 | `POST /api/leads/{id}/stage` | `lead_stage_history` |
| 客户时间线 | `GET /api/leads/{id}/timeline` | 多表聚合 |
| 活动报名 | `POST /api/events/{id}/registrations` | `event_registration` |
| 报告快照 | `POST /api/reports` | `report_snapshot` |

## 7. 角色任务完整性排查

客户增长实现前必须检查：

| 检查项 | 顾问要求 |
| --- | --- |
| 创建数据 | 可手动新建线索 |
| 上传资料 | 可上传或粘贴客户资料 |
| 触发 AI/规则 | 可触发客户研判 |
| 查看推荐 | 可查看推荐项目 |
| 更新状态 | 可更新客户阶段 |
| 写入跟进 | 可新增跟进记录 |
| 创建待办 | 可创建和完成任务 |
| 进入详情 | 可进入客户 360 |
| 数据落表 | 所有动作对应数据库表 |
| API 对接 | 所有动作有 API 规划 |

## 8. UI 约束

1. 顾问页采用高密度业务处理布局。
2. 顾问主操作区放在首屏，不用说明性卡片挤占空间。
3. 客户列表和漏斗避免大面积空白。
4. 客户 360 采用清晰 tabs，不把所有内容堆在单页。
5. 无权限入口不显示。
6. UI 效果图阶段至少输出 4 套整体风格方案，客户增长页必须分别展示这些风格下的首屏信息密度和主操作位置。

## 9. 验收标准

1. 顾问进入客户增长后可新建线索。
2. 新建线索后可研判、推荐、跟进和更新阶段。
3. 客户 360 能聚合画像、推荐、咨询、跟进、活动和报告。
4. 管理者看到经营指标，不被顾问操作表单干扰。
5. 不出现“二期助手”或无权限占位。
