# 企业官网门户与角色权限后台信息架构设计

## 1. 背景

本设计取代此前“角色工作台 + 二期助手”结构。系统入口仍为公开官网门户，但登录后不再套统一角色工作台，也不再把企业助手、学生助手收纳为“二期助手”。所有后台入口按客户需求表和真实角色权限组织。

## 2. 总体结构

```text
公开官网门户 -> 登录入口 -> 按权限进入对应业务后台
```

| 层级 | 用户 | 目标 |
| --- | --- | --- |
| 公开官网门户 | 游客、潜在客户、家长、学生、合作方 | 建立信任、解释服务、项目活动展示、客服 Agent、咨询报名 |
| 登录入口 | 所有后台用户 | 演示角色进入；生产认证后续接入 |
| 业务后台 | 顾问、员工、老师、学生、管理者、管理员 | 按权限完成实际业务任务 |

## 3. 公开官网门户

官网页面：

- 首页
- 企业介绍
- 业务服务
- 项目/课程
- 活动/讲座
- 知识/FAQ + 客服 Agent
- 联系咨询
- 登录

官网必须覆盖客服 Agent 需求：

1. 公司信息咨询。
2. 公司业务查询。
3. 海外留学政策查询。
4. 课程与项目推荐。
5. 活动与讲座报名。
6. 常见问题自助解答。
7. 日常轻量闲聊。

官网禁止展示：

- CRM 客户列表。
- 客户 360。
- 员工日报。
- 学生心理预警明细。
- 审计日志。
- 权限矩阵。
- OpenAPI。
- seed。
- 接口健康和内部 fallback 调试信息。

## 4. 登录入口

登录页只承担连接官网与后台的任务。当前可使用演示角色进入，但文案必须极简。

建议保留一句：`当前为演示登录，真实认证后续接入。`

| 角色 | 默认入口 |
| --- | --- |
| 顾问 | 客户增长 |
| 员工 | 员工工作台 |
| 老师 | 学生服务工作台 |
| 学生 | 学生服务台 |
| 管理者 | 经营管理后台 |
| 管理员 | 系统治理 |

## 5. 权限展示规则

1. 无权限一级入口直接不渲染。
2. 不显示“当前角色隐藏”“角色暂不展示”或禁用菜单占位。
3. 页面内无权限动作优先不展示。
4. 只有在业务上下文必须保留时，才用简短只读状态说明。
5. 系统治理页可展示权限配置，但只对管理员开放，管理者可按需只读。

## 6. 登录后后台

### 6.1 顾问：客户增长

顾问后台是高频业务处理页面，必须包含：

- 新建客户线索。
- 上传或粘贴客户资料。
- 触发客户研判。
- 查看推荐项目。
- 客户列表和漏斗。
- 创建跟进记录。
- 创建待办任务。
- 更新客户阶段。
- 进入客户 360。

### 6.2 员工：员工工作台

员工后台面向快捷操作：

- 客户快捷录入。
- 客户查询。
- 客户状态更新。
- 口述日报。
- 组织架构查询。
- 新人指南问答。
- 受控只读查询。

### 6.3 老师：学生服务工作台

老师后台面向待办处理：

- 请假待审批。
- 反馈待处理。
- 心理辅助预警待跟进。
- 学业节点查看。
- 申请进度查看。
- 学生相关报告入口。

### 6.4 学生：学生服务台

学生后台面向自助服务：

- 请假申请。
- 投诉建议。
- 申请进度查询。
- 学业考务查询。
- 海外生活支持问答。
- 心理倾诉入口。
- 升学项目推荐。

### 6.5 管理者：经营管理后台

管理者后台面向指标、报告和风险：

- 增长总览。
- 客户经营分析报告。
- 员工日报日/周汇总。
- 学生心理健康周报。
- 投诉处理周报。
- 风险队列。

### 6.6 管理员：系统治理

管理员后台面向系统治理：

- 用户、角色、权限。
- 审计日志。
- 通知中心。
- 知识来源。
- OpenAPI。
- seed。
- fallback 和接口状态。

## 7. UI 约束

1. 删除解释性长文。
2. 只保留必要标题、字段、按钮、状态、错误、风险提示和空状态。
3. 不在业务页面展示阶段规划、技术解释或内部演示说明。
4. 各角色后台按功能数量合理排布，不强行统一布局。
5. 功能少的角色使用紧凑入口；功能多的角色使用分区、表格和队列。
6. 避免大面积空白、右侧长列表堆叠和禁用导航占位。
7. UI 效果图阶段至少输出 4 套整体风格方案：简约大气、科技高级感、绚烂花哨、稳重政企；用户确认最终风格后再进入实现。

## 8. 数据闭环要求

每个入口必须追踪到：

- 角色。
- 前端页面。
- API 分组。
- 数据库表。
- 验收动作。

顾问“手动录入客户线索”是强制检查项，必须在客户增长页作为主操作出现。

## 9. 验收标准

1. 未登录默认进入官网。
2. 官网客服 Agent 覆盖客户需求表公开咨询场景。
3. 登录后不同角色进入不同业务后台。
4. 无权限入口不展示。
5. 不再出现“二期助手”作为一级入口。
6. 顾问可手动录入线索。
7. 前端文案精简，布局不出现明显空白。
