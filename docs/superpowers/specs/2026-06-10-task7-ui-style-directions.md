# Task 7 UI 风格效果图方案

本文件对应 `docs/superpowers/plans/2026-06-10-role-permission-full-system-refactor.md` 的 Task 7 方案阶段。

预览产物：

- `frontend/design-previews/task7-style-directions.html`

当前仅输出效果图方向，不把任何一套风格写入正式前端路由或 `frontend/src` 页面实现。用户确认最终风格后，再进入正式实现、截图验证、构建和提交。

## 方案 A：简约大气

适用场景：长期办公、信息录入、客户队列和学生服务高频使用。

优点：留白克制、信息层级清楚、学习成本低，适合顾问、员工、老师日常使用。

缺点：展示冲击力较弱，官网首屏和路演场景不够抓眼。

对官网影响：官网更像可信服务门户，强调咨询、项目、活动和登录入口。

对后台影响：后台保持低噪音，适合长时间处理客户、日报、审批和报告。

## 方案 B：科技高级感

适用场景：突出 AI 研判、智能报告、Dify/fallback、受控查询等能力。

优点：数据感更强，对比更明确，适合演示智能能力。

缺点：深色或强对比界面需要控制阅读疲劳，长期办公不宜过度炫技。

对官网影响：官网更强调智能咨询、项目匹配和数据驱动。

对后台影响：管理者、管理员和研判页面更有产品感，但老师/学生页面需要降噪。

## 方案 C：绚烂花哨

适用场景：官网营销、活动报名、路演和短时间展示。

优点：视觉冲击强，活动和项目入口更醒目。

缺点：后台长期使用容易疲劳，也可能降低政企客户的信任感。

对官网影响：官网更像活动型增长入口，咨询和报名转化更突出。

对后台影响：后台必须降低色彩面积，只保留关键操作和状态高亮。

## 方案 D：稳重政企

适用场景：教育服务交付、国企/政企客户、管理后台和审计治理。

优点：可信、克制、表格和状态清晰，适合真实交付。

缺点：视觉记忆点不如科技或绚烂方案。

对官网影响：官网突出可信服务、业务范围、咨询和登录入口。

对后台影响：顾问、员工、老师、管理者、管理员后台可保持统一可信风格，同时按角色任务差异布局。

## 建议

建议选择方案 D「稳重政企」作为正式后台主风格，并吸收方案 A 的留白和信息层级处理。官网可保留更轻的视觉表达，但不应变成后台功能陈列页。
