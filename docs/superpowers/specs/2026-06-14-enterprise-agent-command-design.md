# 企业助手 AI 指挥台专项设计

## 背景

企业助手是当前 V3 生产基础计划中的核心 Agent 能力，不应停留在简单聊天窗口。它服务员工及继承员工工作台能力的顾问、老师、管理者，负责把日常问题转成可确认、可跳转、可追踪的业务动作。

本设计以 `frontend/design-previews/enterprise-agent-status-near-preview.html` 和 `frontend/design-previews/enterprise-agent-status-near-preview.png` 为视觉基准。

## 设计定位

采用 “AI 指挥台 + 官网级产品界面” 的方向：

- 视觉需要有产品级质感，能作为核心能力展示给客户和管理层。
- 布局需要适合员工每天使用，不能为了展示感牺牲操作效率。
- 中央 AI 工作区是最大主画面，参考 Codex 式主工作区优先结构。
- 左侧只做窄导航和位置识别，不放大块场景卡片。
- 快捷入口保留可用性，但降为辅助，不抢 AI 工作区。

## 页面结构

### 1. 全局顶栏

顶栏只保留后台身份、当前视角、待确认数、今日处理数和主要动作。

要求：

- 不做大 hero，避免压缩操作区。
- 文案使用业务语言，不出现 “fallback”“真实 API”“V3” 等实现话术。
- 主按钮可以是 “新建日报草稿” 或当前场景最常用动作。

### 2. 左侧窄导航

左侧窄导航负责全局模块定位，例如快捷录入、日报、组织、客户、新人指南、AI。

要求：

- 宽度保持轻量，不做卡片列表。
- 当前企业助手入口突出，但不影响中央工作区。
- 刷新后仍应通过当前页面 key 恢复到企业助手。

### 3. 中央 AI 指挥台

中央区域是页面主体，由标题区、场景切换、对话流、结构化业务产物和输入框组成。

要求：

- 宽度优先分配给对话流和业务产物。
- 对话不是唯一结果，必须承载结构化草稿、查询结果、跳转动作。
- 场景切换使用轻量 chip：日报、组织、客户、指南。
- 输入框固定在中央工作区底部，符合持续对话习惯。

### 4. 执行队列贴近 AI 结果

执行队列放在中央主区右侧，贴近 AI 产物，而不是放到最外侧。

原因：

- 员工看到草稿或查询结果后，最需要立刻知道当前动作处于哪一步。
- 执行队列是当前 AI 结果的状态反馈，不是普通右侧信息流。

要求：

- 展示识别意图、生成草稿、写入记录、跳转处理等步骤。
- 当前步骤高亮，其他步骤弱化。
- 与当前上下文放在同一区域，例如客户、学生、材料缺口。

### 5. 最右侧辅助栏

最右侧只放辅助内容：快捷启动、最近结果、视觉权重或验收提示。

要求：

- 快捷启动采用轻量按钮，不做大卡片。
- 最近结果用于复盘，不替代执行队列。
- 辅助栏不能抢中央主工作区视觉权重。

## 核心业务闭环

企业助手必须覆盖以下闭环能力：

1. 日报草稿：口述内容 -> 结构化草稿 -> 本人确认 -> 提交日报。
2. 组织查询：事项问题 -> 定位部门/负责人 -> 跳转待办或组织页。
3. 客户受控查询：客户问题 -> 阶段/跟进/下一步建议 -> 跳转客户查询或客户 360。
4. 新人指南：制度/流程问题 -> 知识回答 -> 推荐下一步入口。

AI 输出只能作为建议、结构化输入或草稿；涉及写入的动作必须经过用户确认。

## 视觉原则

- 主色使用深绿和象牙白，保持教育服务品牌信任感。
- 金色只用于场景焦点或轻量高亮，不能大面积铺开。
- 背景可以有细网格和玻璃质感，但不能影响阅读。
- 卡片圆角控制在产品后台可接受范围，避免过度柔软。
- 字号服务于工作区密度，标题克制，不做营销页式大字堆叠。
- 中央 AI 产物卡片的层级最高，快捷入口和最近记录层级低一档。

## 权限与入口

- 员工可见企业助手。
- 顾问、老师、管理者继承员工工作台能力时可见企业助手。
- 学生不可见企业助手。
- 管理员是否展示企业助手应服从系统治理视角，不把管理员做成业务入口合集。

## 验收标准

1. 员工进入后台后能从导航进入企业助手。
2. 企业助手不是单纯聊天框，页面至少包含对话区、结构化业务产物、执行队列、快捷启动、最近结果和当前上下文。
3. 日报场景能生成草稿并展示确认动作。
4. 组织查询场景能展示负责人和跳转动作。
5. 客户查询和新人指南至少具备入口和空/结果状态承接。
6. 普通业务角色页面不展示技术实现话术。
7. 结构检查覆盖导航项、页面映射和角色可见性。
8. 前端 `npm.cmd run build` 通过。

## 后续适用范围

本设计不是单页美化规则，而是后续 Agent 页面设计基准。客服 Agent、学生 Agent 和后台其他 Agent 能力应复用同一信息架构：

- 左侧窄导航负责位置。
- 中央 AI 工作区负责持续对话和业务产物。
- 执行状态贴近 AI 结果。
- 快捷入口降权为辅助。
- 右侧栏放上下文、最近结果和复盘信息。
