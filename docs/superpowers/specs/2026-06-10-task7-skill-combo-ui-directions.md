# Task 7 Skill 组合自由发挥 UI 方向

本文件是 Task 7 的第二版效果图方案。上一版主要验证“四套风格均已输出”，但组件骨架过于统一。本版按不同前端 UI skill 组合分别发挥，重点拉开组件形态、信息层次、字体气质和布局结构。

预览产物：

- `frontend/design-previews/task7-skill-combo-directions.html`
- `docs/superpowers/specs/task7-style-screenshots/task7-skill-combo-directions-full.png`

## 效果图 1：服务策展 Editorial Portal

对应组合：

- `product-design:get-context`
- `product-design:ideate`
- `frontend-design`

擅长点：先从产品语境和用户目标出发，再做官网、登录和业务入口的叙事型视觉。适合公开官网、服务可信度展示、咨询转化。

设计特征：编辑式大标题、票据式角色入口、出版物气质字体、少卡片、多叙事段落。

## 效果图 2：智能运营 Command Center

对应组合：

- `frontend-design`
- `impeccable craft`
- `browser verification`

擅长点：把复杂后台做成强数据感的运营指挥舱，突出 AI 研判、风险、报告、系统治理能力。

设计特征：深色指挥舱、左侧符号 rail、热区网格、右侧任务雷达、强状态对比。

## 效果图 3：可信台账 Governance Ledger

对应组合：

- `design-taste-frontend`
- `frontend-design`
- `verification-before-completion`

擅长点：克制、可信、反模板，适合权限、审计、日报、投诉、审批等证据链页面。

设计特征：台账式导航、表格优先、低装饰、强边界和审计感。

## 效果图 4：任务动线 Service Flow OS

对应组合：

- `impeccable craft`
- `gsap motion thinking`
- `frontend-design`

擅长点：把跨角色任务闭环变成可演示的流程动线，适合后续增加轻动效、转场和流程讲解。

设计特征：横向业务流、步骤节点、柔和空间、强调“从官网到报告”的闭环。

## 效果图 5：空间画布 Workbench Canvas

对应组合：

- `image-to-code mindset`
- `frontend-design`
- `browser screenshot QA`

擅长点：从视觉目标反推页面结构，适合客户 360、管理复盘、系统排障这类多信息块协同页面。

设计特征：网格画布、浮动业务块、非线性布局、每个角色的信息块独立但可关联。

## 当前建议

如果目标是“真实交付 + 高级感”，建议从效果图 3「可信台账 Governance Ledger」作为后台基底，再融合效果图 2 的经营管理仪表和效果图 1 的官网首屏。这样官网有品牌感，后台有秩序，管理/治理页面有可信度。
