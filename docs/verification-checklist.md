# V3 最终验收清单

## 1. 使用边界

本清单用于最终 B1-B12 验收准备，执行依据仍是 `docs/business-flow-test-plan.md` 和 `docs/test-object-claim-table.md`。

假设：

- 不执行破坏性数据库操作。
- 不要求真实外部账号或密钥；真实 Dify、真实 MySQL 只在对应配置存在时验收。
- 验收开始前统一初始化一次演示数据，开始后不再中途重置。

取舍：

- 自动化验证用于证明代码、文档、接口合同和构建状态。
- 人工浏览器验收用于证明跨角色业务闭环、URL 恢复、布局和业务文案。
- SQLite 验收与 MySQL 验收必须分开记录，不能把 SQLite 通过写成 MySQL 通过。
- Dify 未配置 fallback 与真实 Dify 验收必须分开记录；fallback 通过不等于真实 Dify 数据集命中通过。

成功标准：

- B1-B12 每条链路都有负责人、对象编号、入口 URL、自动化证据、人工验收证据和数据库口径。
- 登录、官网、顾问、CRM、员工、学生、老师、报告、权限、审计、通知、Dify、导出、部署均有验收记录。
- 阻断和高优先级缺陷清零后，才能进入最终交付结论。

## 2. 验证类型

| 类型 | 覆盖内容 | 必须记录 |
| --- | --- | --- |
| 自动化验证 | `pytest`、前端 build、文档覆盖检查、专项目标脚本 | 命令、运行时间、结果、失败摘要 |
| 人工浏览器验收 | 登录、官网、角色后台、B1-B12 跨角色动线、URL 恢复、布局和文案 | 角色、入口 URL、对象编号、截图或说明 |
| SQLite 验收 | 本地演示库写入、读取、刷新持久化、跨表关联 | 数据库类型写明为 SQLite，记录对象 ID |
| MySQL 验收 | MySQL 服务、Alembic head、空库建表、生产配置连接、同链路抽查 | 数据库类型写明为 MySQL，记录库名、迁移版本和对象 ID |

## 3. B1-B12 执行矩阵

| 编号 | 链路 | 自动化验证 | 人工浏览器验收 | 数据库口径 |
| --- | --- | --- | --- | --- |
| B1 | 官网活动报名到顾问承接 | 后端活动/报名接口测试，前端 build | 游客在官网报名，顾问查看报名或线索承接 | SQLite 记录报名和线索；MySQL 另行记录同链路结果 |
| B2 | 顾问新建线索到客户 360 | CRM API 和权限测试 | 顾问新建线索，进入队列和客户 360 | 线索、客户状态和时间线刷新后仍存在 |
| B3 | 顾问跟进到管理者增长 | 报告/增长接口测试 | 顾问新增跟进、任务、阶段变更，管理者看增长总览 | 跟进、任务、阶段历史和管理汇总可关联 |
| B4 | 员工快捷录入到顾问/管理者 | 企业助手和客户写入权限测试 | 员工提交客户快捷录入或状态更新，顾问/管理者复核 | 员工写入不形成孤立记录 |
| B5 | 员工日报到管理者日报 | 日报 API 和报告测试 | 员工提交日报，管理者查看日报汇总 | 日报列表、汇总报告和审计记录一致 |
| B6 | 学生请假到老师审批 | 学生服务权限和审批测试 | 学生提交请假，老师同意或驳回，学生查看状态 | 请假状态、处理记录和通知可追踪 |
| B7 | 学生反馈到老师处理 | 反馈接口和通知测试 | 学生提交反馈，老师处理，学生查看处理记录 | 工单状态、回复记录和通知一致 |
| B8 | 老师成绩录入到学生查询 | 成绩 API 和数据范围测试 | 老师录入或修改成绩，学生查询 | 成绩只对对应学生可见，刷新后存在 |
| B9 | 老师心理辅助预警到管理者周报 | 心理预警和报告测试 | 老师记录跟进，管理者看心理周报 | 只记录辅助识别和跟进，不出现诊断结论 |
| B10 | 报告生成到详情查看 | 报告生成、详情、导出测试 | 管理者生成报告，打开详情，刷新后恢复 | 报告快照、审计和导出记录一致 |
| B11 | 活动创建到官网报名 | 活动 API 和公开页构建检查 | 后台新建活动，官网报名，后台名单更新 | 活动、报名和签到口径可关联 |
| B12 | 项目维护到顾问推荐 | 项目/研判接口测试 | 后台维护项目，顾问做研判或推荐 | 推荐结果能引用最新项目方向 |

## 4. 专项覆盖

| 范围 | 验收要求 |
| --- | --- |
| 登录 | 覆盖管理员、管理者、顾问、员工、老师、学生、测试账号；退出后不残留后台内容。 |
| 官网 | 不展示 CRM、审计、权限、系统状态等内部信息；咨询、报名、项目和登录入口可用。 |
| 顾问 | 手动录入线索、CRM、客户 360、研判、跟进、任务和活动邀约闭环。 |
| CRM | 搜索、筛选、详情、状态流转、时间线、任务、活动报名同步。 |
| 员工 | 快捷录入、日报、组织架构、新人指南、受控客户查询。 |
| 学生 | 请假、反馈、成绩、进度、考务、生活支持和心理关怀边界。 |
| 老师 | 请假审批、反馈处理、心理预警跟进、成绩录入和学生服务队列。 |
| 报告 | 生成、历史、详情、解释、导出 PDF/Word 和刷新恢复。 |
| 权限 | 前端不展示无权限入口，后端关键接口拒绝越权访问。 |
| 审计 | 创建、审批、报告生成、导出、通知处理和 Agent 确认动作可追踪。 |
| 通知 | 已读、处理完成、打开业务对象和跨角色待办回显可用。 |
| Dify | Dify 未配置 fallback 不阻断主业务；真实 Dify 验收需记录 key/app/dataset 配置、命中场景和失败重试。 |
| 导出 | 报告 PDF/Word 按权限导出，文件名、content type、审计记录一致。 |
| 部署 | 后端启动、前端启动、环境变量、CORS、健康检查、生产初始化、备份恢复按 runbook 验收。 |

## 5. 最终记录模板

| 字段 | 记录要求 |
| --- | --- |
| 验收编号 | 使用 B1-B12、SYS、U、P、S、C、K、E、T、ST、M、A、O 编号。 |
| 验收类型 | 自动化验证 / 人工浏览器验收 / SQLite 验收 / MySQL 验收。 |
| 角色与入口 | 记录角色、账号、入口 URL 和当前页面。 |
| 对象编号 | 记录客户、学生、活动、报告、通知、审计或导出 ID。 |
| Dify 状态 | 记录未配置 fallback / 真实 Dify 验收 / 不适用。 |
| 结果 | 通过 / 不通过 / 有条件通过。 |
| 证据 | 命令输出、截图、对象 ID、数据库连接类型或失败摘要。 |
